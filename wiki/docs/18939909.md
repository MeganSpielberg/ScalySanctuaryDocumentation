# Devlog: Animated Gecko

> ℹ️ **Note:** Author: Megan

## 💡 Intro

A game about caring for a living animal needs that animal to feel
alive. The early version of Scaly Sanctuary had a static sprite of a
leopard gecko sitting in the terrarium - it served its purpose as a
placeholder, but it undermined the core premise of the game. If the
research study was measuring engagement and knowledge acquisition
through a caretaking experience, a motionless image on a screen was not
going to convincingly simulate that experience. Players needed to feel
like they were responsible for something that moved, reacted, and
existed in the space. This devlog documents how that problem was solved
through a collaboration with a 3D artist and a two-layer animation
architecture built in [Godot](Godot_851969.md).

## 🖼️ Initial Implementation

The starting point was a hand-drawn static sprite placed in the
terrarium scene. It looked presentable enough in isolation, but once the
terrarium itself became more detailed, the contrast between the richly
illustrated environment and the inert gecko became increasingly
obvious. No idle movement, no reactions, no sense of a creature going
about its day. For a game whose study design depends in part on player
engagement scores, this was a problem worth solving properly.

## 🎨 3D Artist Collaboration

The solution came from a suggestion by Dogfood Studios: SintLucas
Eindhoven, a creative arts school, places student interns at Fontys who
are available to take on projects. Together with project partner
Valentino Clerx, the decision was made to reach out, and 3D artist Lucas
Gubbels agreed to take on the gecko model.

The collaboration was iterative. The first rounds of feedback focused on
anatomical accuracy - limb proportions, toe count, body shape - making
sure the model would hold up against players who actually know what a
leopard gecko looks like. Once the geometry was right, attention shifted
to shading, getting the visual style to match the illustrated
environments already in the game rather than looking like a 3D asset
dropped into a 2D scene. The final animations were exported as sprite
sheets and imported into Godot as an `AnimatedSprite2D` node, keeping
the integration lightweight and compatible with the existing 2D
pipeline.

## ⚙️ Implementation

With the assets in place, the challenge was making the gecko behave
believably without the logic becoming unmanageable. The solution was a
clean separation into two loosely coupled layers: a logic layer that
decides what the gecko is doing, and a visual layer that decides how
that looks on screen.

The logic layer is a LimboAI Hierarchical State Machine (HSM) managing
four core states: Idle, Walking, Sleeping, and Eating. Each state is its
own script that decides when to transition. `idle_state.gd`, for
example, waits for a minimum idle duration and for the gecko's current
animation loop to complete before randomly dispatching to Walking (75%)
or Sleeping (25%). This detail matters: transitions only fire at clean
animation boundaries, so the gecko never snaps mid-blink into a new
state. `walking_state.gd` similarly waits until the gecko has reached
its walk target before dispatching, and `eating_state.gd` requires three
full loops of the eat animation before returning to Idle.

```gdscript
func _update(_delta: float) -> void:
    idle_duration += _delta
    check_timer += _delta
    if check_timer >= CHECK_INTERVAL:
        check_timer = 0.0
        if idle_duration < MIN_IDLE_DURATION:
            return
        if not _has_completed_idle_loop():
            return  # wait for the gecko's idle animation to finish a loop

        if randf() < TRANSITION_CHANCE:
            if randf() < 0.75:
                get_parent().dispatch("to_walking")
            else:
                get_parent().dispatch("to_sleeping")
```

The visual layer is handled entirely by `gecko_model.gd`, an
`AnimatedSprite2D` that receives the current state name from the HSM
each frame via `set_pet_state()` and translates it into the correct
animation, movement, and facing direction. The two layers communicate
through a single `StringName` — `pet.state` — plus a small set of
read-back signals that let the logic layer query visual progress without
coupling to animation internals. The HSM never directly touches the
`AnimatedSprite2D`, and the gecko model never reads the game state.

Walking movement is handled inside `_update_walking()`: each frame the
gecko steps toward a randomly chosen point within the terrarium's walk
polygon, with speed easing in and out through `walk_enter` and
`walk_exit` transition animations. If the target requires the gecko to
turn around, a turn animation plays first. `Geometry2D.is_point_in_polygon()`
on the gecko's feet anchor point keeps it from wandering outside the
floor area.

The eating state feeds back into the UI through
`get_eat_animation_progress_ratio()`, which exposes how far through the
three-loop eat animation the gecko currently is. `terrarium_view.gd`
uses this to progressively hide food bowl sprite slots in sync with the
animation, so the bowl visually empties as the gecko eats rather than
snapping to empty when the state ends.

## 🪞 Reflection

The gap between a static sprite and a gecko that idles, wanders, falls
asleep, and eats in response to player actions is significant, not just
visually, but in terms of what the game is asking the player to feel.
Caretaking implies responsibility for something that has ongoing needs
and responds to whether those needs are met. The animated gecko makes
that feedback loop tangible in a way the static sprite never could.
Given that the study measures both knowledge gain and engagement, having
a protagonist that actually feels present in the terrarium was not a
cosmetic concern but a foundational one. The collaboration with Lucas
Gubbels, arranged through Valentino and me, and made possible by
Dogfood Studios' suggestion, delivered an asset that elevated the game's
credibility as both a product and a research instrument. During
playtesting, some people noticed the geckos' behavior and responded
positively to it by making comments on its cuteness and high quality.
