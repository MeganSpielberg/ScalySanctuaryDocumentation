# Godot

## 💡 Why Godot

Scaly Sanctuary was built in **Godot 4.6**. Godot was chosen for a
combination of practical reasons:

- **Familiarity:** Megan already had prior experience with the engine,
  which removed a significant ramp-up cost given the project's limited
  development time.

- **Free and open-source:** No licensing cost or revenue-share
  concerns, which matters for a student research project with no
  budget.

- **Lightweight:** Godot runs comfortably on modest hardware, which was
  important since not all development and testing machines (including
  laptops used during data collection events) had high-end specs.

- **Fit for scope:** The project's scope - a 2D caretaking game with a
  handful of interactive scenes - matched well with Godot's strengths
  in 2D development, without the overhead of a heavier engine like
  Unreal or even Unity.

## ⚙️ How It Was Used

Several Godot-specific features were leveraged throughout development,
documented in more detail in their respective devlogs:

| Feature | Used For |
|----|----|
| `AnimatedSprite2D` | Driving the gecko's sprite-sheet animations (idle, walking, sleeping, eating) — see [Devlog: Animated Gecko](18939909.md) |
| LimboAI (Hierarchical State Machine plugin) | Modeling both the original pet behavior state machine and the gecko's logic layer — see [Devlog: Animal State Machine](884764.md) and [Devlog: Animated Gecko](18939909.md) |
| `HTTPRequest` | Asynchronous calls to the LLM gateway and backup provider — see [Devlog: LLM Host Failsafe](3899400.md) |
| `CapsuleShape2D` + manual hit-testing | Body-zone detection in the petting/handling scene — see [Devlog: Reptile Handling](18874369.md) |
| `Geometry2D.is_point_in_polygon()` | Constraining gecko walk targets to the terrarium floor area |

A full list of custom and third-party Godot plugins built or used for
the project is documented separately in [Godot Plugins](Godot-Plugins_4325378.md).
